# 风格轮动图表呈现对齐修改文档

## 目的

这份文档只处理一件事：

把当前第一阶段 Web 版的图表展示方式，尽量对齐到旧项目 `frontend/src/views/quant/StyleRotation.vue` 的呈现逻辑。

重点不是改后端算法，而是改前端图表表现、图表细节和信息传达方式。

## 强约束

这次修改的目标不是“风格接近”，而是：

- 数据呈现方式尽量一致
- 鼠标悬停 tooltip 的信息结构尽量一致
- 图表内部视觉层级尽量一致
- 图例、颜色、线型、缩放、联动方式尽量一致

允许不完全一样的部分只有：

1. 页面外壳布局
2. 页面标题文案
3. 当前版本为了支持任意两个标的而做的文字泛化

不允许自由发挥的部分：

1. 图层结构
2. tooltip 组织方式
3. 图表联动方式
4. 主图中的红绿区域表达
5. 主线、均线、参考线、信号点的呈现逻辑

## 参考基准

当前运行中的第一阶段页面：

- `http://localhost:5175/`

旧项目参考实现：

- `frontend/src/views/quant/StyleRotation.vue`

## 当前页面与目标的核心差异

### 1. 当前版本是三张独立图卡

当前 `5175` 实现：

- 收益价差图、滚动收益图、净值图分成三个独立图表实例
- 每张图单独一张卡片
- 图表之间没有联动缩放，也没有统一游标

旧项目目标：

- 一个 ECharts 实例内放三层 grid
- 三张图共用同一条时间轴
- 鼠标悬停、缩放、拖动应联动

这会直接影响阅读体验。

旧项目更适合看风格轮动，因为三张图是同一时间坐标下联动观察的。

### 2. 当前版本的价差图语义和旧项目不一致

当前 `5175` 实现：

- `spread` 是一条金色线
- `ma` 是一条绿色虚线
- `p90_dynamic` / `p10_dynamic` 被画成两条动态序列线
- `buy/sell` 用 `markPoint` 打在主线上

旧项目目标：

- `spread` 是黑色主线
- `MA20` 是橙色虚线
- `p90` / `p10` 用全局参考虚线显示
- 价差正负区域要有明显红绿背景区分
- 信号更强调“箭头语义”，而不是普通点标记

这里最大的视觉差异是：

- 当前版本强调动态分位数曲线
- 旧项目强调“价差位置 + 趋势 + 高低阈值 + 风格强弱区域”

## 目标呈现方式

### 图表整体结构

必须改成：

- 单个 ECharts 实例
- 三层 grid
- 三组 xAxis / yAxis
- `axisPointer.link` 联动
- `dataZoom` 同时控制三张图

结构参考旧项目：

1. 第 1 层：收益价差图
2. 第 2 层：左右标的滚动收益率对比
3. 第 3 层：左右标的归一化净值对比

并尽量对齐旧项目的布局参数：

- `grid[0]`: `top: '5%'`, `height: '28%'`, `left: '5%'`, `right: '5%'`
- `grid[1]`: `top: '38%'`, `height: '25%'`, `left: '5%'`, `right: '5%'`
- `grid[2]`: `top: '68%'`, `height: '22%'`, `left: '5%'`, `right: '5%'`

x 轴结构应尽量对齐旧项目：

- 第 1 层 xAxis 不显示
- 第 2 层 xAxis 不显示
- 第 3 层 xAxis 显示

y 轴结构应尽量对齐旧项目：

- 第 1 层：`收益差值(%)`
- 第 2 层：`年滚动收益(%)`
- 第 3 层：`归一化净值`

### 图 1：收益价差图

应改成下面的表现方式：

1. 增加 `spread > 0` 的红色填充区域
2. 增加 `spread < 0` 的绿色填充区域
3. `spread` 主线改为黑色或深灰色细线
4. `ma` 改为橙色虚线
5. 不再默认把 `p90_dynamic` / `p10_dynamic` 当成整条动态曲线强调展示
6. 主图上优先展示全局参考线：
   - `global_p90`
   - `global_p10`
7. 信号标记改成更接近旧项目的箭头语义：
   - `buy` 用绿色向上箭头
   - `sell` 用红色向下箭头

更具体的要求：

- 主图 series 顺序应尽量与旧项目一致
- 先画正区间填充
- 再画负区间填充
- 再画 `spread`
- 再画 `MA`
- 最后画信号

填充区域要求：

- `spread > 0`：红色半透明区域
- `spread < 0`：绿色半透明区域
- 两条区域线本身不应有可见描边

主线要求：

- `spread` 使用深色细线
- `MA` 使用橙色虚线
- `global_p90` 使用红色虚线
- `global_p10` 使用绿色虚线

如果当前后端只有 `summary.global_p90/global_p10`，前端必须直接使用，不要自己重算。

### 图 2：滚动收益率对比

应改成下面的表现方式：

1. 左侧标的收益线用蓝色
2. 右侧标的收益线用橙色
3. 增加 `yAxis = 0` 的参考线
4. 尽量弱化装饰，突出对比关系

细节要求：

- 左右收益率都用实线
- 不显示点位 symbol
- 线宽要比主图的 `spread` 稍粗一点或相当
- `yAxis = 0` 参考线使用深灰虚线

### 图 3：归一化净值对比

应改成下面的表现方式：

1. 左侧标的净值线用蓝色
2. 右侧标的净值线用橙色
3. `yAxis` 开启 `scale: true`
4. 保持“起点统一为 1”这件事的视觉直觉

细节要求：

- 不显示点位 symbol
- 两条净值线都用实线
- 颜色逻辑与图 2 保持一致，不要重新换色系

## 当前版本需要修改的具体点

### A. 不再使用三个独立 chart 实例

当前问题：

- `spreadChart`
- `returnChart`
- `navChart`

是三个实例。

修改要求：

- 合并成一个 `chartInstance`
- 单个 option 内完成三层图

### B. 主图不要再把 `p90_dynamic` / `p10_dynamic` 作为重点可视化主角

说明：

后端计算仍然保留动态分位数，用于信号判断，这个不改。

但展示层要更接近旧项目：

- 信号判定继续使用动态分位数
- 图上主要展示 `global_p90` / `global_p10` 作为参考线

如果要保留动态分位数，也只能作为可选增强，不应抢占主视觉。

### C. 恢复红绿风格强弱区域

必须补上：

- `spread > 0` 时红色区域填充
- `spread < 0` 时绿色区域填充

这在旧项目里是核心信息表达，不是装饰。

它直接告诉用户：

- 左侧风格更强
- 右侧风格更强

### D. 信号标记要改成箭头语义

当前问题：

- 只是 `markPoint`
- 标签写 `BUY` / `SELL`

修改要求：

- `buy` 使用绿色向上箭头
- `sell` 使用红色向下箭头
- 标签可以简化，不必大面积文字块遮挡曲线

### E. tooltip 要恢复成旧项目那种统一信息面板

当前问题：

- 三张图分别 tooltip
- 数值风格不完全统一

修改要求：

- 使用统一联动 tooltip
- 同一个日期下，三层图同时响应
- 百分比字段统一显示 `xx.xx%`
- 净值字段统一显示 `x.xxxx`

必须尽量复刻旧项目 tooltip 的组织方式：

1. 第一行显示日期
2. 后面逐行显示当前鼠标下所有 series
3. 每行结构为：
   - 左侧彩色圆点 + 系列名
   - 右侧格式化后的数值
4. tooltip 背景应接近旧项目的浅色半透明面板

数值格式化规则必须固定：

- 如果系列名包含 `收益` 或 `价差`，显示 `toFixed(2) + '%'`
- 其他数值显示 `toFixed(4)`

颜色小圆点规则：

- 如果颜色是渐变对象，取第一个色阶颜色
- 否则直接使用 series 颜色

tooltip formatter 的目标效果要尽量接近旧项目：

```js
formatter: (params) => {
  let html = `<div>${params[0].axisValue}</div>`
  params.forEach(item => {
    if (item.seriesName) {
      let val = item.value
      if (item.seriesName.includes('收益') || item.seriesName.includes('价差')) {
        val = val.toFixed(2) + '%'
      } else {
        val = val.toFixed(4)
      }
      const color = item.color?.colorStops ? item.color.colorStops[0].color : item.color
      const marker = `<span style="display:inline-block;margin-right:4px;border-radius:10px;width:10px;height:10px;background-color:${color};"></span>`
      html += `<div style="display:flex;justify-content:space-between;width:220px;">
        <span>${marker}${item.seriesName}</span>
        <span style="font-weight:bold">${val}</span>
      </div>`
    }
  })
  return html
}
```

允许字段名根据当前项目做泛化，但展示结构和格式化逻辑不要变。

### F. 加回 dataZoom 联动

当前问题：

- 当前三图没有统一滑动缩放

修改要求：

- 使用一个 slider dataZoom
- 使用 inside dataZoom
- 三层图联动缩放

参数建议尽量贴近旧项目：

```js
dataZoom: [
  { type: 'slider', xAxisIndex: [0, 1, 2], bottom: 10, start: 0, end: 100 },
  { type: 'inside', xAxisIndex: [0, 1, 2] }
]
```

还要补上：

```js
axisPointer: { link: { xAxisIndex: 'all' } }
```

这样鼠标悬停时三层图才能严格联动。

## 系列命名要求

为了兼容“任意两个标的”能力，系列名允许泛化，但语义要与旧项目一致。

推荐改成：

- `价差>0(左侧强)`
- `价差<0(右侧强)`
- `收益价差`
- `MA20`
- `${left_name || left_symbol}收益率`
- `${right_name || right_symbol}收益率`
- `${left_name || left_symbol}净值`
- `${right_name || right_symbol}净值`

注意：

- 旧项目里写的是“小盘强/大盘强”
- 新项目不能写死成“小盘/大盘”
- 但表达逻辑必须保持一致

## 交互细节要求

必须补齐下面这些旧项目式交互细节：

1. 鼠标移动时三层图同位置联动
2. 拖动缩放时三层图同时间区间联动
3. tooltip 在任何一层触发，信息都按统一逻辑显示
4. 图表 resize 时只需要 resize 一个实例
5. 不要再用三个实例分别维护

## 实现边界

这次修改只改前端图表呈现。

不应改动：

1. 后端 pandas 计算逻辑
2. `/api/style-rotation` 返回结构
3. `/api/instruments` 和 `/api/market-data/sync` 的接口契约

如果确实缺少某个字段：

- 优先从现有 `summary` 和 `series` 中组合使用
- 不要先改后端，除非完全无法对齐旧项目表现

## 后端字段建议

为了对齐旧项目体验，前端展示层建议优先使用这些字段：

- `series.dates`
- `series.spread`
- `series.ma`
- `series.left_return`
- `series.right_return`
- `series.left_nav`
- `series.right_nav`
- `signals`
- `summary.global_p90`
- `summary.global_p10`

如果当前后端没有直接返回：

- `global_p90`
- `global_p10`

那就继续从 `summary` 里取，不要前端自己重算。

## 允许保留的当前版本优点

以下内容可以保留，不需要因为对齐旧项目而回退：

1. 左右标的自由选择
2. 同步来源选择
3. 单页整体布局
4. 摘要卡片
5. 当前更现代的页面配色和外壳样式

也就是说：

- 外层页面风格可以是新的
- 图表内部表达逻辑要向旧项目靠拢

## 修改优先级

### P0

必须先改：

1. 三图合并为一个联动 chart
2. 恢复红绿价差区域
3. `spread` / `MA` / `global_p90` / `global_p10` 的旧项目式表达
4. 信号改成箭头语义
5. 加上 dataZoom 联动

### P1

然后再改：

1. tooltip 细节格式
2. 颜色和线宽更贴近旧项目
3. 图例名称更中文化、业务化

### P2

可选增强：

1. 增加“使用手册”弹窗
2. 增加快捷时间区间
3. 增加图例开关

## 验收标准

改完后应满足：

1. 页面仍然是单页，不引入后台系统
2. 图表仍然支持任意两个标的组合
3. 三层图必须共用时间轴
4. 鼠标悬停时三层图联动
5. 缩放和拖动时三层图联动
6. 主图必须有红绿区域表达风格强弱
7. 主图必须显示 `spread`、`MA`、`global_p90`、`global_p10`
8. `buy/sell` 必须是箭头语义，不是普通点
9. 第二层和第三层图的配色逻辑要和旧项目一致
10. tooltip 的日期行、数据行结构、数值格式化规则要和旧项目尽量一致
11. 只保留一个 chart 实例，不再有三个独立实例
12. 页面外层可以保留新风格，但图表内部交互必须接近旧项目

## 结论

当前 `5175` 的问题不在“有没有功能”，而在“图表表达方式偏离旧项目语义”。

需要改的是图表层，而不是接口层。

外层页面可以保留当前风格，但图表内部应明显向旧项目对齐。
