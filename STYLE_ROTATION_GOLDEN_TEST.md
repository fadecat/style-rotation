# 风格轮动黄金测试样例

## 目的

这份文档不是业务说明，而是实现校验样例。

新项目中的 pandas 计算逻辑写完后，必须能跑出这里的结果。

如果跑不出来，通常说明以下某个环节写偏了：

- 日期对齐方式不对
- `pct_change` 窗口不对
- `dropna` 时机不对
- `expanding().quantile()` 写错成了别的
- 信号生成顺序不对
- 切片后净值重算方式不对

## 测试目标

验证这几个点：

1. 任意 `left_symbol:right_symbol` 组合都能计算
2. 计算必须先基于完整区间，再按展示区间切片
3. `buy/sell` 信号必须正确生成
4. `left_nav/right_nav` 必须在展示区间起点重算为 `1`

## 测试参数

这是一个用于单元测试的小样例，不使用生产默认窗口。

请求参数：

```json
{
  "left_symbol": "AAA",
  "right_symbol": "BBB",
  "start_date": "2024-01-04",
  "end_date": "2024-01-09",
  "return_window": 1,
  "ma_window": 2,
  "quantile_window_min": 2
}
```

说明：

- 这里故意用很小的窗口，方便人工校验
- 真实生产环境仍然建议 `250 / 20 / 20`

## 完整计算过程

### 第 1 步：按交易日对齐

对齐后数据：

| trade_date | left_close | right_close |
|---|---:|---:|
| 2024-01-02 | 100.00000 | 100.00000 |
| 2024-01-03 | 101.00000 | 100.00000 |
| 2024-01-04 | 106.05000 | 100.00000 |
| 2024-01-05 | 106.05000 | 100.00000 |
| 2024-01-08 | 101.80800 | 100.00000 |
| 2024-01-09 | 103.84416 | 100.00000 |

### 第 2 步：计算滚动收益

这里 `return_window = 1`。

公式：

```text
left_return_t = (left_close_t / left_close_{t-1} - 1) * 100
right_return_t = (right_close_t / right_close_{t-1} - 1) * 100
spread_t = left_return_t - right_return_t
```

得到完整有效区间：

| trade_date | left_return | right_return | spread |
|---|---:|---:|---:|
| 2024-01-03 | 1.00 | 0.00 | 1.00 |
| 2024-01-04 | 5.00 | 0.00 | 5.00 |
| 2024-01-05 | 0.00 | 0.00 | 0.00 |
| 2024-01-08 | -4.00 | 0.00 | -4.00 |
| 2024-01-09 | 2.00 | 0.00 | 2.00 |

### 第 3 步：计算均线和动态分位数

这里：

- `ma_window = 2`
- `quantile_window_min = 2`

得到：

| trade_date | spread | ma | p90_dynamic | p10_dynamic |
|---|---:|---:|---:|---:|
| 2024-01-03 | 1.00 | NaN | NaN | NaN |
| 2024-01-04 | 5.00 | 3.00 | 4.60 | 1.40 |
| 2024-01-05 | 0.00 | 2.50 | 4.20 | 0.20 |
| 2024-01-08 | -4.00 | -2.00 | 3.80 | -2.80 |
| 2024-01-09 | 2.00 | -1.00 | 3.80 | -2.40 |

### 第 4 步：生成完整区间信号

信号判定规则：

- `sell`：上一日 `spread > ma`，当日 `spread < ma`，且上一日 `spread > p90_dynamic`
- `buy`：上一日 `spread < ma`，当日 `spread > ma`，且上一日 `spread < p10_dynamic`

按完整区间计算，必须得到：

```json
[
  {
    "date": "2024-01-05",
    "type": "sell",
    "spread": 0.0
  },
  {
    "date": "2024-01-09",
    "type": "buy",
    "spread": 2.0
  }
]
```

### 第 5 步：按展示区间切片

请求里的展示区间是：

- `start_date = 2024-01-04`
- `end_date = 2024-01-09`

所以最终返回给前端的 `series` 只保留：

| trade_date | left_close | right_close | left_return | right_return | spread | ma | p90_dynamic | p10_dynamic |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 2024-01-04 | 106.05000 | 100.00000 | 5.00 | 0.00 | 5.00 | 3.00 | 4.60 | 1.40 |
| 2024-01-05 | 106.05000 | 100.00000 | 0.00 | 0.00 | 0.00 | 2.50 | 4.20 | 0.20 |
| 2024-01-08 | 101.80800 | 100.00000 | -4.00 | 0.00 | -4.00 | -2.00 | 3.80 | -2.80 |
| 2024-01-09 | 103.84416 | 100.00000 | 2.00 | 0.00 | 2.00 | -1.00 | 3.80 | -2.40 |

### 第 6 步：切片后重算归一化净值

展示区间的第一天是 `2024-01-04`，所以：

- `left_base = 106.05000`
- `right_base = 100.00000`

最终：

| trade_date | left_nav | right_nav |
|---|---:|---:|
| 2024-01-04 | 1.0000 | 1.0000 |
| 2024-01-05 | 1.0000 | 1.0000 |
| 2024-01-08 | 0.9600 | 1.0000 |
| 2024-01-09 | 0.9792 | 1.0000 |

## 这个样例能卡住哪些错误实现

如果实现有下面这些问题，通常无法通过这个样例：

1. 先按 `start_date` 切片再计算
2. 把动态分位数写成全局固定分位数
3. 把动态分位数写成 rolling quantile
4. 净值没有在展示区间起点重算
5. `sell/buy` 的判定使用了当日分位数而不是上一日分位数
6. `left_symbol/right_symbol` 逻辑写死成固定标的

## 使用建议

你开新 AI 会话时，除了给它规格文档，还应该附上这份黄金样例，并明确要求：

1. 先写后端计算函数
2. 用这份样例做自测
3. 跑通后再写前端页面

这样复刻准确度会明显更高。
